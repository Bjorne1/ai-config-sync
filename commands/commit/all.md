# 提交所有修改
提交当前工作区中所有未提交的代码变更并推送。

## Git 提交规范：
使用约定式格式：<type>(<scope>): <主题>，其中 type = feat|fix|docs|style|refactor|test|chore|perf。主题：最多50字符，使用祈使语气（如"添加"而非"已添加"）。简单改动：仅需一行提交信息。复杂改动：添加正文说明改动内容和原因（每行72字符）。如涉及不同功能，应拆分为多个提交。除了专业术语外，使用中文注释。
